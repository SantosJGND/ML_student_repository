# Machine Learning Engineering Course Repository

Welcome to the ML Engineering course project repository.

This repository is structured around a single evolving machine learning system that you will build step by step throughout the course.

---

## 🧭 Course Philosophy

This is not a notebook-only course.

You will build a real ML system that evolves across:

- Data engineering
- Model training
- Experimentation
- Deployment
- Monitoring

Each session adds a new layer to the same system.

---

## 🧱 Repository Structure

```

data/ → raw and processed data (to be filled during course)
src/ → core ML pipeline code
notebooks/ → exploration and experiments
pipelines/ → future ML workflow orchestration
models/ → trained model artifacts
deployment/ → serving system (later sessions)
monitoring/ → drift and performance tracking (later sessions)
configs/ → configuration files
tests/ → validation and correctness checks
docs/ → project documentation
```

---

## 🚀 How to Use This Repo

Each session you will:

1. Modify existing code (do not create separate projects)
2. Implement new components
3. Extend the same ML pipeline
4. Document your changes

---

## 🧠 Key Principle

> You are building one system, not multiple assignments.

Everything you do should integrate into a single evolving ML pipeline.

---

## 🧪 Starting Point

Begin with:

```
src/train.py
notebooks/01_exploration.ipynb
docs/case_study.md
```

---

## 📌 Rules

- Keep code modular
- Avoid hardcoding paths
- Document your assumptions
- Always think in terms of production systems, not just models

---

## 📈 End Goal

By the end of the course you will have:

- A full ML system
- A reproducible training pipeline
- A deployable model
- A monitoring strategy
- A structured engineering repository

---

## 👨‍🏫 Note

This repository evolves with the course. Do not expect it to be complete at the beginning.
